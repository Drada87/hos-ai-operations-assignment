# HOS AI Operations Assignment

This repository contains my submission for the HOS AI Operations / Chief of Staff technical assignment. The project is divided into three independent tasks, each addressing a different aspect of problem-solving, system design, and implementation.

---

## Repository Structure

### 📘 [01. Company Second Brain Strategy](01_Company_Second_Brain/)

A practical knowledge management strategy designed to centralize organizational knowledge, reduce dependency on individuals, and enable fast information retrieval through structured documentation, lightweight governance, and AI-assisted search.

**Deliverables**

- Knowledge management strategy
- Information architecture
- 30-day rollout plan
- Architecture diagram

---



### 02. Flight Recommendation Engine

A modular Python application that evaluates and ranks multi-city flight itineraries using weighted business rules and traveler preferences.

**Key Features**

- Modular architecture
- Explainable scoring engine
- Business-rule evaluation
- Recommendation summary
- Tradeoff analysis

---



### 03. Live Air Traffic Dashboard

A Streamlit dashboard that visualizes live aircraft data from the OpenSky Network API.

**Key Features**

- Live flight data
- Interactive dashboard
- KPI metrics
- Search and filtering
- Automatic refresh

---



## Technologies Used

- Python 3.12
- Streamlit
- Pandas
- Plotly
- Requests

---



## Design Principles

Across all three tasks, the project follows the same engineering principles:

- Modular architecture
- Single Responsibility Principle (SRP)
- Separation of concerns
- Readable and maintainable code
- Explainable design decisions
- Low coupling and high cohesion
